## Reflections

### 1. Reflection on Coding in Markdown
**Situation:** PRP372S required me to build a digital portfolio using GitHub and a Markdown language which I was not familiar with, and to also code my entire CV using the markdown language instead of inserting it as a pdf file.
**Task:** I had to structure all my CV content - personal details, education, skills, experience, projects, references - using only GitHub flavoured Markdown so it would render as a clean webpage on GitHub pages.
**Action:** I learnt key syntax like '#' for headings, '**bold**' for experiences, projects etc, '-' for bullet points, and '|' tables to lay out my skills so they were scannable. When the layout looked somehow overcrowded, I also added '---' lines to break up sections and tested each commit to see how it rendered live.
**Result:** I can now build a full CV using only plain text that is version-controlled and mobile friendly. Coding my CV manually gave me more control over formatting than a PDF, and I now do understand how developers write documentation and README files.

### 3. Reflection on the use of GitHub Pages
**Situation:** The brief required a live, public link to be submitted on Teams/ Blackboard and not just a code repository.
**Task:** I had to publish my 'Pertuinia221692568.github.io' repo using GitHub pages so my Markdown-coded CV, embedded video and reflections would display as a live website. At that time I was not that much familiar with GitHub pages.
**Action:** In Settings -> Pages, I set the source to 'Deploy from branch', selected 'main' and '/root', then saved those changes. I learned that the repo name must exactly match the repo's username or else the site will not build. I checked the deployment after 2 minutes and used hard refresh to see every recent change that I make in the page.
**Result:** My Full portfolio is now live. I now understand how to go from code to deployed site using GitHub pages, which means I can now host any future projects or documentation for free and share with recruiters instantly.
